1. Kaun DB se user laata hai?

👉 UserDetailsService

Ye ek interface hai

Tum isko implement karte ho

Ye DB se user fetch karta hai

👉 Example:

@Override
public UserDetails loadUserByUsername(String email) {
    User user = repo.findByEmail(email)
        .orElseThrow(() -> new UsernameNotFoundException("User not found"));

    return new org.springframework.security.core.userdetails.User(
        user.getEmail(),
        user.getPassword(),
        List.of()
    );
}

✔️ So answer:
👉 UserDetailsService DB se user laata hai

🔹 2. Kaun password compare karta hai?

👉 PasswordEncoder

Ye password ko hash karta hai

Aur login ke time compare karta hai

👉 Example:

passwordEncoder.matches(rawPassword, encodedPassword)

✔️ So answer:
👉 PasswordEncoder password compare karta hai

🔹 3. Kaun decide karta hai login success/fail?

👉 AuthenticationManager

Ye pura process control karta hai

Ye internally:

UserDetailsService ko call karta hai

PasswordEncoder ko call karta hai

Aur final decision deta hai

✔️ So answer:
👉 AuthenticationManager decide karta hai login success/fail

🧠 Final Clean Flow (Super Important)

Jab tum login karte ho:

User → Controller → AuthenticationManager
                          ↓
              UserDetailsService (DB check)
                          ↓
              PasswordEncoder (password match)
                          ↓
               AuthenticationManager decision
